# Phase 2 Demo — Git Discipline

## What's new this phase

The project now lives on GitHub with real guardrails: nobody (including the
repo owner) can push straight to `main`, and every change goes through a
reviewable pull request.

## See it yourself

1. Browse **https://github.com/SaintJude/payguard** — note the commit
   history is one clean commit per PR (squash merging), not a pile of WIP
   commits.
2. Try pushing directly to `main` from a local clone — GitHub will refuse
   with `GH006: Protected branch update failed`.
3. Open **https://github.com/SaintJude/payguard/pull/new/main** from a
   branch with changes — the description box pre-fills with the
   Summary/Phase/Test-plan template automatically.
